개인 리뷰

페이지 구성

header로 메뉴를 나누어 줌.

div를 사용하여 영향을 최소화 하고,
클래스로 구분함

footer로 디자인을 기업에 맞게
로고와 경영페이지 및 회사정보를
세분화 함.

반응형 웹 SKILL

media query 로 Mobile,Tablet, PC
영역을 나누어 줌

이미지에 Transform Scale을 사용하여
클릭 부분임을 강조하였으며,
부가설명을 추가

Position relative , absolute
사용하여 퍼블리싱을 하였고,
위치값을 지정

display flex,grid를 활용하여 X축과
Y축을 지정하여 자연스러움을 줌

hover, Scale 사용시 이미지 어둡게 만들기

영역없이 부모요소 하나로 된다고 생각함.

하지만, 영역한개를 더 생성하여 그 값에 대입하여
opacity와 rgba값을 작성하고 translate로 위치를
지정하여 작동시킴

이미지 및 텍스트 넘쳐서 가로 스크롤 해결

처음 하나하나 개발자 도구에서 크기
줄여가며 확인.

그렇게 힘들게 확인하였지만,
넘쳤을 땐 개발자 도구에서 영역을 하나씩
지워가며 금방 찾게 됨.

flex 적용하여 위치 나누기 어려울 때

flex 적용이 안되어서 영역도 나누어보고
flex-derection 흐름이 잘못되었다고
생각하였지만,

grid를 배우고 난 뒤 grid colum, row를
사용하여 적용.

이 웹을 만들면서 media query의 장점이 인상깊었다.

자연스러움 흐름뿐만 아니라 많은 종류의 기기로 봤을 때
사이즈에 맞춰지는 것이 인상깊었으며,

처음부터 끝까지 직접 제작해 볼 수 있는 좋은 기회였고
배운 것을 전부 사용하고 싶은 마음에 과해지기도 했지만

피드백을 통해 과한 부분은 덜어내고 부족한 부분은
채울 수 있는 좋은 경험을 할 수 있었다.
