변경 전

Box-shadow: 0 0 10px rgb(0,0,0)을 적용하였을 때 카드뉴스 뒷부분이 단면적으로 나타났 으며 border값을 준 것처럼 굵게 나타나고 부자연스러웠다.

변경 후

Box-shadow 0 0 24px rgba(0,0,0,0.13) 첫번째 0px의 경우 윗부분 값 두번째 값은 아래부분 값이었다.

세번째는 전체적 뒷부분 값이었던 걸로 보아 원하는 것은 겹쳐지는 효과가 잘 보여야 하 기 때문에 세번째 값을 더 줌.

더 주는 것과 동시에 너무 진하게 보여 rgba로 변 경 후 투명 값을 조절하여 자연스러워짐

잘한점

  • 처음 조원들과 분위기를 이끌어내 활기차게 프로젝트를 이어갔다는 점
  • 적극적으로 참여하여 시간을 효율적으로 사용한 점
  • 모르는 점은 즉각 알려주고 피드백을 해주었다는 점
  • 전보다 CSS에 대한 이해도와 기술적으로 향상되었다는 점

부족한 점

  • JavaScript에 대한 작성방식 이해도가 낮았던 점
  • 팀원들과 Font 설정 통일을 못 한 점
  • CSS와 JavaScript 활용 능력이 떨어졌다는 점

배운점

JavaScript Scroll에 대한 이해도가 올라갔다는 점

디자인에 대한 생각의 폭이 넓어지게 된 점

클래스 값을 쉽게 알아보도록 작성하는 방식을 알게 된 점

웹이 자연스러운 흐름을 갖게끔 노력하며 작성하게 된 점

git organizations을 통해 팀 작업에 수월함을 보였으며, 각자 git을 생성하여 마무리 단계에 합쳐 git을 다시 생성하고 pages를 열었을 때 구현된 것이 좋았다.

분명히 높은 기술들을 구사하는 내용인 것은 맞았지만 구현해내는 과정에서 뿌듯함을 느꼈으며 문제가 생겼을 시에 공통된 클래스 값을 통해 더욱 쉽게 내용을 찾고 상대방이 코드를 열었을 때 즉각 알아볼 수 있었다.

variable 파일을 통해 더욱 쉽게 작업했던 부분도 인상 깊었고 배워가는 점이 많았었으며 클래스 이름에 중요도도 알게 된 프로젝트였다.
